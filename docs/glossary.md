# 词汇表

本词汇表包含教材中所有核心术语的精确定义，按字母顺序排列。

---

## A

#### AI（人工智能）
让机器表现出需要人类智能才能完成的行为的技术集合。包括理解语言、识别图像、做出决策等能力。

**示例**： Siri回答问题、抖音推荐视频、AlphaGo下棋都是AI的应用。

#### API（应用程序编程接口）
软件之间互相通信的约定方式。就像餐厅菜单——你不需要知道厨房如何运作，只需要按菜单点菜。

**示例**： 天气App通过气象局的API获取天气数据。

#### API Key
访问API的身份凭证，相当于"钥匙"。需要保密，不要分享给他人。

---

## B

#### Bash
一种常用的Unix Shell，是许多Linux系统的默认命令行解释器。

---

## C

#### Claude
Anthropic公司开发的AI助手，以安全性和有用性著称。本教材的主要演示工具。

#### Claude Code
Anthropic开发的命令行AI编程助手，可以在终端中与AI交互，处理文件和代码。

#### 上下文窗口（Context Window）
大模型一次能处理的token数量上限。就像人的短期记忆容量。

---

## G

#### Git
分布式版本控制系统，用于跟踪文件的变化历史，多人协作开发必备工具。

#### GitHub
基于Git的代码托管平台，提供代码存储、协作、发布等功能。

---

## H

#### 幻觉（Hallucination）
AI生成看似合理但实际错误或不存在的内容。就像人"一本正经地胡说八道"。

**示例**： AI可能编造不存在的论文引用或历史人物。

#### Homebrew
macOS系统上流行的包管理器，用于安装命令行工具和应用软件。

---

## L

#### LLM（大语言模型）
参数量极大、能理解和生成自然语言的深度学习模型。如GPT-4、Claude等。

---

## N

#### Node.js
可以在服务器端运行JavaScript的平台，许多现代开发工具基于它构建。

#### npm
Node.js的包管理器，用于安装和管理JavaScript库和工具。

---

## O

#### OpenClaw
开源的个人AI助手框架，可以本地部署，支持自定义技能和扩展。

---

## P

#### Prompt
给大模型的输入指令或问题。Prompt的质量直接影响AI回答的质量。

#### 包管理器（Package Manager）
自动化安装、更新、卸载软件的工具。如macOS的Homebrew、Node.js的npm。

#### 路径（Path）
文件在系统中的位置表示。如 `/Users/name/Documents/file.txt`。

---

## S

#### Shell
解释执行命令行指令的程序，是用户与操作系统内核之间的接口。

---

## T

#### Token
大模型处理文本的最小单位，可能是字、词或子词。中英文token计算方式不同。

**示例**： "Hello world" 大约是3个token（Hello + 空格 + world）。

#### Transformer
大模型的基础架构，使用自注意力机制处理序列数据，是现代LLM的核心。

#### 终端（Terminal）
通过命令行与计算机交互的界面，也称为"命令行"或"控制台"。

---

## Y

#### YAML
一种人类可读的配置文件格式，使用缩进表示层级结构，常用于配置文件。

---

## Z

#### 终端（Terminal）
通过文本命令与计算机交互的界面。macOS上可以通过"终端"App打开。

#### 少样本学习（Few-shot Learning）
通过在Prompt中提供几个示例，让模型学习新任务的输入输出模式。

#### 零样本学习（Zero-shot Learning）
模型在没有见过示例的情况下，直接执行新任务的能力。
